# 院线电影订阅

自动发现中国大陆、香港、澳门、台湾院线上映电影，检查 MoviePilot 绑定的 Emby / 媒体库中是否已存在，未存在且未订阅时自动添加电影订阅。

处理流程：

1. 按配置的地区和日期范围调用 TMDB 电影发现接口。
2. 限定院线发行类型：默认只接受 `3` 院线上映；如开启“包含点映/小规模院线”，才同时接受 `2`。
3. 拉取电影详情，并按配置的类型限制过滤，例如只订阅“动画/动漫”。
4. 用发行日期详情再次确认 CN/HK/MO/TW 中至少一个地区的原始院线上映日期在日期范围内，并默认忽略复映/重映发行记录。
5. 按 TMDB ID 检查 Emby / 媒体库是否已存在。
6. 已存在则跳过；订阅已存在也跳过。
7. 不存在且未订阅时，使用 MoviePilot 原生 `SubscribeChain.add` 添加电影订阅。

配置说明：

- 执行周期：5 位 cron 表达式，默认每天 09:00。
- 上映地区：默认 `CN`、`HK`、`MO`、`TW`。
- 订阅类型限制：按 TMDB 电影类型过滤，支持动作、冒险、动画/动漫、喜剧、犯罪、纪录、剧情、家庭、奇幻、历史、恐怖、音乐、悬疑、爱情、科幻、惊悚、战争、西部等；留空表示不限制。
- 开始日期 / 结束日期：用于历史补订；留空时按回溯天数和未来天数自动计算。
- 回溯天数 / 未来天数：未填日期时生效，默认前后 30 天。
- 每地区页数：每个地区从 TMDB 拉取多少页，建议 3，最大 20。
- 最低评价数 / 最低热度：用于粗略过滤冷门或数据噪声，0 表示不限制。
- 包含/排除：支持正则表达式，匹配标题、原标题、别名、简介、标语；默认排除常见网络电影/流媒体首映词。
- 包含点映/小规模院线：关闭时只接受正式院线上映，尽量排除流媒体/网络发行；开启时也接受点映、小规模院线发行。
- 排除复映/重映：默认开启；同一地区有多条院线发行记录时，会先忽略备注包含重映/复映/修复/周年/Re-release 等关键词的发行记录，再只使用该地区最早的有效院线上映日期参与日期范围判断。
- Emby服务器：可选择一个或多个已配置 Emby；留空时使用 MP 默认媒体库存在性判断。

注意：

- 媒体库存在性依赖 MoviePilot 媒体服务器同步数据，建议先确认 Emby 已配置并同步过媒体库。
- 若要补订过往院线电影，可填写明确的开始日期和结束日期后点“立即运行一次”。
- 插件会记录已存在、已订阅、新增订阅的历史，历史页显示海报、名称、订阅类型、上映地区、上映时间和加入时间；订阅类型会带题材，如 `电影-动漫`、`电影-喜剧、家庭`；缺失海报会自动按 TMDB ID 补取，点击海报可放大查看。如需重新跑同一批数据，开启“清理历史记录”后再立即运行。
