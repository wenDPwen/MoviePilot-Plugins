# RSS资源择优下载

从一个或多个 RSS 地址中读取影视资源，使用 MoviePilot 原生识别能力匹配媒体信息，再按 TMDB ID、季、集归组。

处理流程：

1. 拉取 RSS 并读取标题、简介、下载链接、页面链接、大小、发布时间。
2. 按包含、排除、种子大小规则过滤候选资源。
3. 调用 MoviePilot 媒体识别，拿到 TMDB ID 等元数据。
4. 按同一影视内容归组，默认每组只选择 RSS 顺序中的第一个。
5. 检查已绑定的 Emby 媒体库是否存在对应 TMDB 内容，已存在则跳过。
6. 未存在时推送给默认下载器，或配置中指定的下载器与保存目录。

配置说明：

- RSS地址：每行一个。
- 包含/排除：支持正则表达式，匹配标题和描述。
- 种子大小：单位 GB，支持 `3` 或 `1-50`。
- 使用择优规则：开启后按选择策略在同组候选中选种子；关闭时每组默认只取 RSS 顺序第一个。
- 选择策略：RSS顺序第一个、优先大体积、优先小体积。
- 使用订阅优先级规则：复用 MoviePilot 中配置的订阅过滤规则进一步过滤候选资源。
- Emby服务器：可选择一个或多个已配置 Emby；留空时使用 MoviePilot 默认媒体库存在性判断。
- 下载器：留空使用默认下载器。
- 保存目录：留空使用下载器默认目录。

注意：

- 媒体库存在性依赖 MoviePilot 的媒体服务器同步与 `media_exists` 能力。建议先确认 Emby 已在 MoviePilot 中配置并完成同步。
- 插件会把已下载或已存在的媒体组写入历史，避免后续重复处理。如需重新处理，打开“清理历史记录”后立即运行一次。
